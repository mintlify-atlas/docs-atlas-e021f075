## Overview

Management views provide functionality for managers (gerentes) to oversee and manage colonies (colonias) in the system. These views include dashboards, listing, creation, editing, and deletion capabilities with permission checks.

### Methods

#### get_queryset()

Implements filtering and search functionality.

```python
def get_queryset(self):
    # Optimize related queries
    qs = Colonia.objects.prefetch_related("distritos").all()
    
    # Get filter parameters
    q = self.request.GET.get("q")
    estado = self.request.GET.get("estado")
    distrito = self.request.GET.get("distrito")
    
    # Apply filters
    if q:
        qs = qs.filter(nombre__icontains=q)
    if estado:
        qs = qs.filter(estado=estado)
    if distrito:
        qs = qs.filter(distritos__id=distrito)
    
    return qs.distinct()
```

**Source:** `gerencia/views.py:30-41`

### Query Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `q` | string | Search term for colony name (case-insensitive) |
| `estado` | string | Filter by colony status |
| `distrito` | int | Filter by district ID |
| `page` | int | Page number for pagination (25 items per page) |

### Request/Response Examples

<CodeGroup>
```bash GET List All
curl -X GET http://example.com/gerencia/colonias/ \
  -H "Cookie: sessionid=abc123"
```

```bash GET With Filters
curl -X GET "http://example.com/gerencia/colonias/?q=san&estado=ACTIVO&distrito=5" \
  -H "Cookie: sessionid=abc123"
```

```bash GET Paginated
curl -X GET "http://example.com/gerencia/colonias/?page=2" \
  -H "Cookie: sessionid=abc123"
```
</CodeGroup>

### Performance Optimization

- Uses `prefetch_related("distritos")` to reduce database queries
- Applies `distinct()` to prevent duplicate results from ManyToMany joins
- Paginated at 25 items per page

#### form_valid()

Handles colony creation with automatic code assignment and district association.

```python
def form_valid(self, form):
    response = super().form_valid(form)
    
    # Find the smallest missing code number
    codigos_existentes = set(
        Colonia.objects.values_list('codigo', flat=True))
    codigo = 1
    while codigo in codigos_existentes:
        codigo += 1
    form.instance.codigo = codigo  # Assign first available number
    
    response = super().form_valid(form)
    
    # If coming from specific district, ensure it's assigned
    distrito_id = self.request.POST.get('distritos')
    if distrito_id:
        try:
            distrito = Distrito.objects.get(id=distrito_id)
            self.object.distritos.add(distrito)
        except Distrito.DoesNotExist:
            pass
    
    messages.success(
        self.request,
        f'Colonia "{self.object.nombre}" creada exitosamente!'
    )
    return response
```

**Source:** `core/views.py:307-333`

### Automatic Code Assignment

The system automatically assigns the lowest available colony code:

1. Query all existing codes
2. Start from code `1`
3. Increment until finding an unused code
4. Assign to new colony

## editar_colonia()

Edits an existing colony's information.

### Function Definition

```python
def editar_colonia(request, colonia_id):
    colonia = get_object_or_404(Colonia, id=colonia_id)
    if request.method == 'POST':
        # Use form for ManyToMany fields
        form = ColoniaForm(request.POST, instance=colonia)
        if form.is_valid():
            form.save()
            messages.success(request, "Colonia actualizada correctamente.")
        else:
            messages.error(request, "Error al actualizar la colonia.")
        return redirect('gerencia:listar_colonias')
    return redirect('gerencia:listar_colonias')
```

## eliminar_colonia()

Deletes a colony if it has no associated requests or surveys.

### Function Definition

```python
def eliminar_colonia(request, colonia_id):
    colonia = get_object_or_404(Colonia, id=colonia_id)
    
    # Check for associated records
    if colonia.solicitudes.exists() or colonia.relevamientos.exists():
        messages.error(
            request,
            "Esta Colonia tiene solicitudes o relevamientos asociados y no puede ser eliminada."
        )
    else:
        colonia.delete()
        messages.success(request, "Colonia eliminada correctamente.")
    
    return redirect('gerencia:listar_colonias')
```

**Source:** `core/views.py:357-367`

### Authentication

- **Required:** Yes (via URL configuration)
- **Permissions:** Delete colonia

### Protection Rules

<Warning>
  Colonies with associated **solicitudes** (requests) or **relevamientos** (surveys) cannot be deleted. This prevents data integrity issues.
</Warning>

## Filtering and Search

The colony list view supports multiple filtering options that can be combined.

### Filter Combinations

```python
# Text search only
?q=san

# Status filter only
?estado=ACTIVO

# District filter only
?distrito=5

# Combined filters
?q=san&estado=ACTIVO&distrito=5

# With pagination
?q=san&page=2
```

### Query Optimization

Filtering is optimized using:

- **Prefetch Related:** Reduces N+1 queries for districts
- **Case-Insensitive Search:** Uses `icontains` for text search
- **Distinct Results:** Prevents duplicates from ManyToMany joins
