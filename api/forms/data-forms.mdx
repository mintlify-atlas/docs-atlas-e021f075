# Data Forms

Forms for managing geographic divisions (Departamentos, Distritos, Colonias) and administrative data in the Gestión de Tierras system.

## Validation Functions

Reusable validation functions used across data forms.

### validar_nombre_letras

Validates names containing only letters, spaces, and apostrophes.

```python core/forms.py:10-18
def validar_nombre_letras(nombre):
    """
    Permite letras, espacios y tildes.
    Rechaza números y símbolos especiales.
    """
    patron = r"^[A-Za-zÁÉÍÓÚáéíóúÑñ\s']+$"
    if not re.match(patron, nombre):
        raise ValidationError("El nombre solo puede contener letras, espacios y apóstrofes.")
    return nombre.strip().upper()
```

**Usage**: Departamento names

### validar_nombre_general

Validates names containing letters, numbers, spaces, and apostrophes.

```python core/forms.py:20-28
def validar_nombre_general(nombre):
    """
    Permite letras, números, espacios y tildes.
    Para Distrito y Colonia.
    """
    patron = r"^[A-Za-z0-9ÁÉÍÓÚáéíóúÑñ\s']+$"
    if not re.match(patron, nombre):
        raise ValidationError("El nombre solo puede contener letras, números, espacios y apóstrofes.")
    return nombre.strip().upper()
```

**Usage**: Distrito and Colonia names

### validar_codigo_numerico

Validates numeric codes are positive integers.

```python core/forms.py:30-36
def validar_codigo_numerico(codigo):
    """
    Valida que el código sea numérico y positivo.
    """
    if codigo is None or codigo < 0:
        raise ValidationError("El código debe ser un número positivo.")
    return codigo
```

### Clean Methods

#### clean_nombre

Validates department name uniqueness (case-insensitive):

```python core/forms.py:57-61
def clean_nombre(self):
    nombre = validar_nombre_letras(self.cleaned_data.get('nombre', ''))
    if Departamento.objects.filter(nombre__iexact=nombre).exclude(id=self.instance.id).exists():
        raise ValidationError(f'El Departamento "{nombre}" ya existe.')
    return nombre
```

- Applies `validar_nombre_letras` (letters, spaces, apostrophes only)
- Converts to uppercase
- Checks uniqueness across all departments
- Excludes current instance when editing

#### clean_codigo

Validates department code uniqueness:

```python core/forms.py:63-67
def clean_codigo(self):

        raise ValidationError(f'El código "{codigo}" ya está asignado a otro departamento.')
```

- Ensures code is positive number
- Checks uniqueness globally
- Excludes current instance when editing

#### clean_codigo

Validates code uniqueness **within the same department**:

```python core/forms.py:89-107
def clean_codigo(self):
    codigo = self.cleaned_data.get('codigo')
    departamento = self.cleaned_data.get('departamento')

    if codigo is None or codigo < 1:
        raise forms.ValidationError("El código debe ser un número positivo.")

    # Validación: único solo dentro del mismo departamento
    if codigo and departamento:
        existe_codigo = Distrito.objects.filter(
            codigo=codigo,
            departamento=departamento
        ).exclude(id=self.instance.id).exists()

        if existe_codigo:
            raise forms.ValidationError(
                f'El código {codigo} ya está asignado en el departamento "{departamento}".')
    return codigo
```

<Note>
  Codes only need to be unique **within each department**, not globally. Different departments can have districts with the same code.
</Note>

#### clean (Cross-field validation)

Validates combinations of nombre+departamento and codigo+departamento:

```python core/forms.py:109-139
def clean(self):
    """
    Validaciones cruzadas: nombre + departamento y código + departamento.
    """
    cleaned_data = super().clean()
    nombre = cleaned_data.get('nombre')
    codigo = cleaned_data.get('codigo')
    departamento = cleaned_data.get('departamento')

    if not departamento:
        raise forms.ValidationError("Debe seleccionar un departamento.")

    # Verificar nombre único por departamento
    if nombre and departamento:
        existe_nombre = Distrito.objects.filter(
            nombre__iexact=nombre.strip(),
            departamento=departamento
        ).exclude(id=self.instance.id).exists()
        if existe_nombre:
            self.add_error('nombre', f'Ya existe un distrito llamado "{nombre}" en el departamento "{departamento}".')

    # Verificar código único por departamento
    if codigo and departamento:
        existe_codigo = Distrito.objects.filter(
            codigo=codigo,
            departamento=departamento
        ).exclude(id=self.instance.id).exists()
        if existe_codigo:
            self.add_error('codigo', f'El código {codigo} ya está asignado en el departamento "{departamento}".')

    return cleaned_data
```

### Model-Level Validation

The form's `clean()` method triggers model validation:

```python core/forms.py:191-200
def clean(self):
    cleaned = super().clean()
    try:
        self.instance.colonia = cleaned.get('colonia')
        self.instance.tipo = cleaned.get('tipo')
        self.instance.observaciones = cleaned.get('observaciones')
        self.instance.full_clean(exclude=None)
    except forms.ValidationError as e:
        raise forms.ValidationError(e.messages)
    return cleaned
```

<Note>
  This approach delegates validation logic to the Solicitud model's `clean()` method, ensuring consistency between form and model-level validation.
</Note>

### Usage Example

```python
from core.forms import SolicitudForm

@login_required
def create_solicitud(request):
    if request.method == 'POST':
        form = SolicitudForm(request.POST)
        if form.is_valid():
            solicitud = form.save(commit=False)
            solicitud.usuario = request.user
            solicitud.save()
            messages.success(request, 'Solicitud creada exitosamente.')
            return redirect('solicitud_detail', pk=solicitud.pk)
    else:
        form = SolicitudForm()
    
    return render(request, 'core/solicitud_form.html', {'form': form})
```

## Form Validation Summary

### Validation Hierarchy

1. **Field-level validation** (`clean_<fieldname>` methods)
2. **Cross-field validation** (`clean()` method)
3. **Model validation** (`model.full_clean()`)

### Common Validation Patterns

#### Uniqueness Validation

```python
if Model.objects.filter(field=value).exclude(id=self.instance.id).exists():
    raise ValidationError(f'Value "{value}" already exists.')
```

#### Scoped Uniqueness

```python
# Unique within a parent (e.g., distrito code per departamento)
if Model.objects.filter(
    code=code,
    parent=parent
).exclude(id=self.instance.id).exists():
    raise ValidationError('Code already exists in this parent.')
```

#### Regex Pattern Validation

```python
import re
patron = r"^[A-Za-zÁÉÍÓÚáéíóúÑñ\s']+$"
if not re.match(patron, value):
    raise ValidationError("Invalid format.")
```
