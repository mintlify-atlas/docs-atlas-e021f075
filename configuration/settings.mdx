## Authentication

### Custom User Model

The system uses a custom user model (`gestion_de_tierras/settings.py:104`):

```python
AUTH_USER_MODEL = 'administrador.User'
```

<Warning>
This setting **cannot be changed** after running initial migrations. The custom user model must be defined in the `administrador` app.
</Warning>

### Authentication URLs

Configure login/logout behavior (`gestion_de_tierras/settings.py:142-144`):

```python
LOGIN_URL = 'login'
LOGIN_REDIRECT_URL = '/'
LOGOUT_REDIRECT_URL = 'logout'
```

- `LOGIN_URL`: Redirect destination for unauthenticated users
- `LOGIN_REDIRECT_URL`: Default redirect after successful login
- `LOGOUT_REDIRECT_URL`: Redirect destination after logout

### Password Validation

The current configuration uses minimal validation for development (`gestion_de_tierras/settings.py:110-137`):

```python
AUTH_PASSWORD_VALIDATORS = [
    {
        'NAME': 'django.contrib.auth.password_validation.MinimumLengthValidator',
        'OPTIONS': {
            'min_length': 4,  # Only 4+ characters required
        }
    },
]
```

<Warning>
**For production**, enable all password validators to ensure strong passwords:

```python
AUTH_PASSWORD_VALIDATORS = [
    {
        'NAME': 'django.contrib.auth.password_validation.UserAttributeSimilarityValidator',
    },
    {
        'NAME': 'django.contrib.auth.password_validation.MinimumLengthValidator',
        'OPTIONS': {
            'min_length': 12,  # Increase to 12+ characters
        }
    },
    {
        'NAME': 'django.contrib.auth.password_validation.CommonPasswordValidator',
    },
    {
        'NAME': 'django.contrib.auth.password_validation.NumericPasswordValidator',
    },
]
```
</Warning>

## Static Files

Static file configuration for CSS, JavaScript, and images (`gestion_de_tierras/settings.py:158-166`):

```python
STATIC_URL = 'static/'
STATICFILES_DIRS = [
    os.path.join(BASE_DIR, 'static'),
]
STATIC_ROOT = os.path.join(BASE_DIR, "staticfiles")
```

### Static Files in Development

- `STATIC_URL`: URL prefix for static files (`/static/`)
- `STATICFILES_DIRS`: Additional locations for static files
- Django automatically serves static files when `DEBUG = True`

### Static Files in Production

- `STATIC_ROOT`: Destination directory for `collectstatic` command
- Run `python manage.py collectstatic` to collect all static files
- Configure your web server (Nginx/Apache) to serve from `STATIC_ROOT`

## Media Files

Configuration for user-uploaded files (`gestion_de_tierras/settings.py:163-164`):

```python
MEDIA_URL = '/media/'
MEDIA_ROOT = os.path.join(BASE_DIR, 'media')
```

- `MEDIA_URL`: URL prefix for uploaded media
- `MEDIA_ROOT`: Filesystem path where media files are stored

<Warning>
In production:
1. Configure web server to serve media files from `MEDIA_ROOT`
2. Set appropriate file permissions (readable by web server, writable by Django)
3. Consider using cloud storage (S3, Google Cloud Storage) for scalability
4. Implement virus scanning for uploaded files
</Warning>

```python

EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
EMAIL_HOST = 'smtp.gmail.com'
EMAIL_PORT = 587
EMAIL_USE_TLS = True
EMAIL_HOST_USER = os.environ.get('EMAIL_USER')
EMAIL_HOST_PASSWORD = os.environ.get('EMAIL_PASSWORD')
DEFAULT_FROM_EMAIL = 'noreply@gestiondetierras.com'
```

## Security Settings

### Development Settings

Current configuration (`gestion_de_tierras/settings.py:24-29`):


<Warning>
**These settings are ONLY for development and are insecure for production!**

- The `SECRET_KEY` is publicly visible

- Empty `ALLOWED_HOSTS` won't work in production

## Environment-Specific Settings

For managing different environments, consider creating separate settings files:

```
gestion_de_tierras/
  settings/
    __init__.py
    base.py      # Common settings
    dev.py       # Development overrides
    prod.py      # Production overrides
    test.py      # Testing overrides
```

### Example: base.py

```python
from pathlib import Path
import os

BASE_DIR = Path(__file__).resolve().parent.parent.parent

# Common settings shared across all environments
INSTALLED_APPS = [...]
MIDDLEWARE = [...]
# etc.
```

### Example: prod.py

```python
from .base import *

DEBUG = False
ALLOWED_HOSTS = os.environ.get('ALLOWED_HOSTS', '').split(',')
SECRET_KEY = os.environ.get('DJANGO_SECRET_KEY')

# Production-specific settings
SECURE_SSL_REDIRECT = True
```

Then set: `export DJANGO_SETTINGS_MODULE=gestion_de_tierras.settings.prod`
