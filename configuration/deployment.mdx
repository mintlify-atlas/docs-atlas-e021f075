## Overview

This guide covers everything needed to deploy Gestión de Tierras to a production environment securely and efficiently.

<Warning>
**Critical**: The default `settings.py` is configured for development only. Follow this guide carefully to secure your production deployment.
</Warning>

## Pre-Deployment Checklist

Before deploying, ensure you have:

- [ ] Production server with Ubuntu/Debian or similar
- [ ] PostgreSQL database configured
- [ ] Domain name with DNS configured
- [ ] SSL certificate (Let's Encrypt recommended)
- [ ] Environment variables management strategy
- [ ] Backup and monitoring plan

### 2. DEBUG Mode

Disable debug mode (`gestion_de_tierras/settings.py:27`):

```python
# Development
DEBUG = True  # ❌ NEVER in production!

# Production
DEBUG = False  # ✅ Always in production
```

When `DEBUG = False`, Django will not serve static files or display detailed error pages.

### 4. Additional Security Settings

Add these to `settings.py` for production:

```python
# HTTPS/SSL Settings
SECURE_SSL_REDIRECT = True  # Redirect all HTTP to HTTPS
SESSION_COOKIE_SECURE = True  # Only send session cookie over HTTPS
CSRF_COOKIE_SECURE = True  # Only send CSRF cookie over HTTPS

# Security Headers
SECURE_BROWSER_XSS_FILTER = True
SECURE_CONTENT_TYPE_NOSNIFF = True
X_FRAME_OPTIONS = 'DENY'  # Prevent clickjacking

# HTTP Strict Transport Security (HSTS)
SECURE_HSTS_SECONDS = 31536000  # 1 year
SECURE_HSTS_INCLUDE_SUBDOMAINS = True
SECURE_HSTS_PRELOAD = True

# Proxy Security (if behind reverse proxy)
SECURE_PROXY_SSL_HEADER = ('HTTP_X_FORWARDED_PROTO', 'https')
```

## Environment Variables

### Using python-dotenv

Install python-dotenv:

```bash
pip install python-dotenv
```

Create `.env` file in project root:

```bash
# .env (never commit this file!)
DJANGO_SECRET_KEY=your-generated-secret-key-here
DEBUG=False
ALLOWED_HOSTS=yourdomain.com,www.yourdomain.com

DB_NAME=gestion_de_tierras_bd
DB_USER=gestion_user
DB_PASSWORD=your-secure-database-password
DB_HOST=localhost
DB_PORT=5432

EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your-email@gmail.com
EMAIL_PASSWORD=your-email-password
```

Load in `settings.py`:

```python
from pathlib import Path
from dotenv import load_dotenv
import os

BASE_DIR = Path(__file__).resolve().parent.parent

# Load environment variables
load_dotenv(os.path.join(BASE_DIR, '.env'))

# Use environment variables
SECRET_KEY = os.environ.get('DJANGO_SECRET_KEY')
DEBUG = os.environ.get('DEBUG', 'False') == 'True'
ALLOWED_HOSTS = os.environ.get('ALLOWED_HOSTS', '').split(',')
```

<Warning>
**Never commit `.env` files to version control!** Add to `.gitignore`:

```gitignore
.env
.env.*
*.env
```
</Warning>

## WSGI Configuration

The WSGI application is defined at `gestion_de_tierras/settings.py:87`:

```python
WSGI_APPLICATION = 'gestion_de_tierras.wsgi.application'
```

### WSGI Server: Gunicorn

Install Gunicorn:

```bash
pip install gunicorn
```

Create `gunicorn_config.py`:

```python
import multiprocessing

bind = "127.0.0.1:8000"
workers = multiprocessing.cpu_count() * 2 + 1
worker_class = "sync"
worker_connections = 1000
max_requests = 1000
max_requests_jitter = 50
timeout = 30
keepalive = 2

# Logging
accesslog = "/var/log/gunicorn/access.log"
errorlog = "/var/log/gunicorn/error.log"
loglevel = "info"

# Process naming
proc_name = "gestion_de_tierras"

# Server mechanics
daemon = False
pidfile = "/var/run/gunicorn.pid"
user = "www-data"
group = "www-data"
```

Run Gunicorn:

```bash
gunicorn gestion_de_tierras.wsgi:application -c gunicorn_config.py
```

### Systemd Service

Create `/etc/systemd/system/gestion-tierras.service`:

```ini
[Unit]
Description=Gestion de Tierras Gunicorn Service
After=network.target postgresql.service

[Service]
Type=notify
User=www-data
Group=www-data
WorkingDirectory=/opt/gestion_de_tierras
Environment="PATH=/opt/gestion_de_tierras/venv/bin"
EnvironmentFile=/opt/gestion_de_tierras/.env
ExecStart=/opt/gestion_de_tierras/venv/bin/gunicorn \
    --config /opt/gestion_de_tierras/gunicorn_config.py \
    gestion_de_tierras.wsgi:application
ExecReload=/bin/kill -s HUP $MAINPID
KillMode=mixed
TimeoutStopSec=5
PrivateTmp=true
Restart=on-failure
RestartSec=5s

[Install]
WantedBy=multi-user.target
```

Enable and start:

```bash
sudo systemctl daemon-reload
sudo systemctl enable gestion-tierras
sudo systemctl start gestion-tierras
sudo systemctl status gestion-tierras
```

## Reverse Proxy: Nginx

Install Nginx:

```bash
sudo apt update
sudo apt install nginx
```

Create `/etc/nginx/sites-available/gestion-tierras`:

```nginx
upstream gestion_app {
    server 127.0.0.1:8000 fail_timeout=0;
}

server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    
    # Redirect HTTP to HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name yourdomain.com www.yourdomain.com;
    
    # SSL Configuration
    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    
    # Security Headers
    add_header X-Frame-Options "DENY" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    
    client_max_body_size 50M;
    
    # Static files
    location /static/ {
        alias /opt/gestion_de_tierras/staticfiles/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
    
    # Media files
    location /media/ {
        alias /opt/gestion_de_tierras/media/;
        expires 7d;
        add_header Cache-Control "public";
    }
    
    # Proxy to Gunicorn
    location / {
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Host $http_host;
        proxy_redirect off;
        proxy_buffering off;
        
        proxy_pass http://gestion_app;
    }
}
```

Enable site:

```bash
sudo ln -s /etc/nginx/sites-available/gestion-tierras /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## Deployment Steps

### 1. Prepare Server

```bash
# Update system
sudo apt update && sudo apt upgrade -y

# Install dependencies
sudo apt install python3 python3-pip python3-venv postgresql nginx

# Create application directory
sudo mkdir -p /opt/gestion_de_tierras
sudo chown $USER:$USER /opt/gestion_de_tierras
```

### 2. Deploy Code

```bash
cd /opt/gestion_de_tierras

# Clone repository (or copy files)
git clone your-repo-url .

# Create virtual environment
python3 -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
pip install gunicorn python-dotenv psycopg2-binary
```

### 3. Configure Environment

```bash
# Create .env file
nano .env
# (Add all environment variables as shown above)

# Set permissions
chmod 600 .env
```

### 4. Database Setup

```bash
# Run migrations
python manage.py migrate

# Create superuser
python manage.py createsuperuser

# Collect static files
python manage.py collectstatic --noinput
```

### 5. Set Permissions

```bash
sudo chown -R www-data:www-data /opt/gestion_de_tierras
sudo chmod -R 755 /opt/gestion_de_tierras
sudo chmod 600 /opt/gestion_de_tierras/.env
```

### 6. Start Services

```bash
# Start Gunicorn
sudo systemctl start gestion-tierras

# Start Nginx
sudo systemctl start nginx

# Check status
sudo systemctl status gestion-tierras
sudo systemctl status nginx
```

### 7. Verify Deployment

Visit `https://yourdomain.com` and verify:
- Site loads correctly
- Static files display
- Login works
- Admin interface accessible
- HTTPS enabled with valid certificate

## Troubleshooting

### 500 Internal Server Error

1. Check Gunicorn logs: `sudo journalctl -u gestion-tierras -n 50`
2. Check Django error log: `tail -f /var/log/gestion_tierras/django_errors.log`
3. Verify environment variables are set correctly
4. Ensure database connection is working

### Static Files Not Loading

1. Run `python manage.py collectstatic`
2. Check Nginx configuration for `/static/` location
3. Verify file permissions: `ls -la /opt/gestion_de_tierras/staticfiles/`
4. Check Nginx error log: `sudo tail -f /var/log/nginx/error.log`

### Database Connection Issues

1. Verify PostgreSQL is running: `sudo systemctl status postgresql`
2. Test connection: `psql -U gestion_user -d gestion_de_tierras_bd -h localhost`
3. Check database settings in `.env`
4. Review PostgreSQL logs: `sudo tail -f /var/log/postgresql/postgresql-*.log`
