## Prerequisites

Before you begin, ensure you have the following installed:

- **Python 3.8+** (Python 3.10 or higher recommended)
- **PostgreSQL 12+** (the system uses PostgreSQL as the database)
- **pip** (Python package installer)
- **Git** (for cloning the repository)
- Basic understanding of Django and command-line operations

<Note>
This quick start guide assumes you're working on a Unix-like system (Linux/macOS). Windows users may need to adjust commands accordingly.
</Note>

## Quick Setup

<Steps>
  <Step title="Clone the Repository">
    Clone the Gestión de Tierras repository to your local machine:
    
    ```bash
    git clone <repository-url>
    cd gestion_de_tierras
    ```
  </Step>

  <Step title="Create Virtual Environment">
    Create and activate a Python virtual environment:
    
    <CodeGroup>
    ```bash Linux/macOS
    python3 -m venv venv
    source venv/bin/activate
    ```
    
    ```bash Windows
    python -m venv venv
    venv\Scripts\activate
    ```
    </CodeGroup>
  </Step>

  <Step title="Install Dependencies">
    Install all required Python packages from `requirements.txt`:
    
    ```bash
    pip install -r requirements.txt
    ```
    
    Key dependencies include:
    - Django 5.0.4
    - psycopg2-binary (PostgreSQL adapter)
    - django-widget-tweaks
    - Pillow (image handling)
    - python-dotenv (environment variables)
  </Step>

  <Step title="Configure Database">
    Create a PostgreSQL database and update the settings:
    
    ```sql
    CREATE DATABASE gestion_de_tierras_bd;
    CREATE USER postgres WITH PASSWORD 'your_password';
    GRANT ALL PRIVILEGES ON DATABASE gestion_de_tierras_bd TO postgres;
    ```
    
    Update database credentials in `gestion_de_tierras/settings.py`:
    
    ```python gestion_de_tierras/settings.py
    DATABASES = {
        'default': {
            'ENGINE': 'django.db.backends.postgresql',
            'NAME': 'gestion_de_tierras_bd',
            'USER': 'postgres',
            'PASSWORD': 'your_password',
            'HOST': 'localhost',
            'PORT': '5432',
        }
    }
    ```
    
    <Warning>
    Never commit database credentials to version control. Use environment variables for production.
    </Warning>
  </Step>

  <Step title="Run Migrations">
    Apply database migrations to create all necessary tables:
    
    ```bash
    python manage.py migrate
    ```
    
    This creates tables for all apps including:
    - core (Departamento, Distrito, Colonia, Solicitud, Relevamiento)
    - administrador (User, Rol)
    - coordinacion, relevamiento, analisis, expedienteo, gerencia, sig, titulacion
  </Step>

  <Step title="Create Superuser">
    Create an administrative superuser account:
    
    ```bash
    python manage.py createsuperuser
    ```
    
    You'll be prompted to enter:
    - Username
    - Email address
    - Password (minimum 4 characters in development)
  </Step>

  <Step title="Collect Static Files">
    Gather all static files (CSS, JavaScript, images):
    
    ```bash
    python manage.py collectstatic --noinput
    ```
    
    This collects files to the `staticfiles/` directory.
  </Step>

  <Step title="Start Development Server">
    Launch the Django development server:
    
    ```bash
    python manage.py runserver
    ```
    
    The server will start at `http://127.0.0.1:8000/`
  </Step>

  <Step title="Access Admin Interface">
    Open your browser and navigate to:
    
    ```
    http://127.0.0.1:8000/admin/
    ```
    
    Log in with the superuser credentials you created.
  </Step>
</Steps>

## Initial Data Setup

Once logged into the admin interface, you should set up:

1. **Roles (Rol)**: Create roles like "Coordinador", "Relevador", "Analista"
2. **Departments (Departamento)**: Add geographic departments
3. **Districts (Distrito)**: Add districts within departments
4. **Colonies (Colonia)**: Register land colonies
5. **Areas and Objectives**: Set up organizational areas and their objectives

<Note>
The system automatically assigns unique codes to Departamentos, Colonias, and Areas when created without explicit codes.
</Note>

## Common Commands

Here are essential Django management commands you'll use frequently:

```bash
# Run development server
python manage.py runserver

# Create database migrations after model changes
python manage.py makemigrations

# Apply migrations
python manage.py migrate

# Create superuser
python manage.py createsuperuser

# Collect static files
python manage.py collectstatic

# Open Django shell
python manage.py shell

# Run tests
python manage.py test
```
